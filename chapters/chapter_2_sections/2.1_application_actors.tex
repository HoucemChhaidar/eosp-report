\section{Application Actors}

The Credix application is designed to serve a hierarchical credit management system with three distinct role levels. Understanding these roles is crucial for defining the specific functional and non-functional requirements of the platform.

\begin{itemize}
    \item \textbf{SUPER\_ADMIN}
    \begin{itemize}
        \item \textbf{Responsibilities:}
        \begin{itemize}
            \item Manage ADMIN users (create, edit, activate/deactivate)
            \item Distribute bulk credit to ADMINs based on wallet transfer amounts
            \item View all ADMINs and their wallets
            \item Monitor system-wide activity
        \end{itemize}
        \item \textbf{Key Features:}
        \begin{itemize}
            \item Admin user management dashboard
            \item Bulk credit transfer to ADMINs
            \item Wallet overview for all managed ADMINs
        \end{itemize}
    \end{itemize}

    \item \textbf{ADMIN}
    \begin{itemize}
        \item \textbf{Responsibilities:}
        \begin{itemize}
            \item Manage USER accounts (employees)
            \item Create and manage user wallets
            \item Set transfer amounts for each user wallet
            \item Distribute bulk credit to users based on transfer amounts
        \end{itemize}
        \item \textbf{Key Features:}
        \begin{itemize}
            \item User management dashboard
            \item Wallet creation and configuration
            \item Bulk credit distribution to employees
            \item Transaction monitoring
        \end{itemize}
    \end{itemize}

    \item \textbf{USER}
    \begin{itemize}
        \item \textbf{Responsibilities:}
        \begin{itemize}
            \item View personal wallet balance
            \item Access transaction history
            \item Make transactions using wallet barcode
            \item Manage personal profile
        \end{itemize}
        \item \textbf{Key Features:}
        \begin{itemize}
            \item Wallet balance display
            \item Transaction history view
            \item Barcode-based payment capability
        \end{itemize}
    \end{itemize}
\end{itemize}
